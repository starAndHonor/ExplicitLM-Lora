## 3. 分析与推断

### 3.1 为什么当前 Loss 更低但 PPL 略高？

```
Loss = -log_prob / num_tokens  (平均)
PPL  = exp(loss_per_token)
```

两种可能：
1. **有效 token 比例不同**：当前实验可能 padding 更少，有效 token 比例更高 → Loss 更低
2. **低频 token 处理差异**：如果某些 token 的 log_prob 特别低，会拉高 PPL 但对平均 Loss 影响较小

### 3.2 训练效率

| 指标 | 参考实验 | 当前实验 |
|------|---------|---------|
| 训练时长/epoch | ~12分钟 | ~12分钟 |
| 总步数 | 1,054,600 | 5,100（当前） |

> 参考实验总步数 1M+，推测是 5 epochs × 200k+ steps/epoch
> 当前实验仅 5,100 步，可能是：
> - batch_size 更小（梯度累积步数更多）
> - 每 epoch 采样步数更少
> - 总共只跑了 1 epoch（但 loss 曲线有 5 个 epoch...）

**待确认**：当前实验实际跑了几个 epoch？总步数 5100 对应多少 epoch？
